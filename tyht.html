🔐 Kanidm — Configuration SSO dans le stack souverain

Version Kanidm : 1.9.x (mars 2026) | OS : Debian 12 + Docker Compose + Caddy 2

1. docker-compose.yml

services:
  kanidm:
    image: kanidm/server:latest
    container_name: kanidm
    restart: unless-stopped
    volumes:
      - kanidm_data:/data
    networks:
      - sovereign_net
    healthcheck:
      test: ["CMD", "kanidmd", "healthcheck", "--config", "/data/server.toml"]
      interval: 30s
      timeout: 10s
      retries: 3

volumes:
  kanidm_data:
    driver: local

2. server.toml (fichier de configuration principal)

version = "2"

bindaddress = "[::]:8443"

domain = "idm.tundomaine.fr"
origin = "https://idm.tundomaine.fr"

trust_x_forward_for = true

db_path = "/data/kanidm.db"

log_level = "info"
[online_backup]
path = "/data/backups/"
schedule = "0 3 * * *"
Important : Le origin doit correspondre exactement à l’URL publique vue par le navigateur via Caddy.

3. Configuration Caddy

idm.tundomaine.fr {
    reverse_proxy kanidm:8443 {
        header_up X-Forwarded-For {remote_host}
        header_up X-Forwarded-Proto {scheme}
        header_up Host {http.request.host}
    }
}

4. Initialisation (première fois)

docker compose up -d kanidm

# Récupérer les comptes admin
docker exec -it kanidm kanidmd recover-account admin
docker exec -it kanidm kanidmd recover-account idm_admin

5. Création des clients OIDC (exemples)

Service Commande principale
Nextcloud kanidm system oauth2 create nextcloud "Nextcloud" https://cloud.tundomaine.fr
Matrix (Dendrite) kanidm system oauth2 create matrix "Matrix" https://chat.tundomaine.fr
Stalwart OIDC natif (UserInfo ou Token Introspection) ou LDAP en fallback
Vaultwarden kanidm system oauth2 create vaultwarden "Vaultwarden" https://pw.tundomaine.fr
Jitsi Meet kanidm system oauth2 create jitsi "Jitsi" https://meet.tundomaine.fr

6. Bonnes pratiques

  • Force WebAuthn + MFA pour tous les utilisateurs
  • Crée un groupe par service (nextcloud_users, matrix_users, etc.)
  • Intègre les backups Kanidm dans Restic
  • Monitor via Prometheus (métriques natives)
  • Avant mise à jour : backup + docker compose pull
