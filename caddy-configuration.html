🚀 Caddy 2 — Reverse Proxy & TLS Automatique

Stack Souverain • Debian 12 • Docker Compose • Kanidm SSO
Version Caddy : 2.9.x (2026)

1. Rôle de Caddy dans ton architecture

Caddy est le point d’entrée unique de ton stack souverain :

  • Terminaison TLS automatique (ACME / Let’s Encrypt)
  • Reverse proxy vers tous les services (Kanidm, Nextcloud, Stalwart, Matrix, Vaultwarden, Jitsi…)
  • Routage par domaine / sous-domaine
  • Rate limiting et protection basique contre les abus
  • Transmission correcte des en-têtes pour Kanidm (SSO)

2. Caddyfile complet recommandé

# ========================
# Caddyfile - Stack Souverain
# ========================

{
    email tonemail@tundomaine.fr
    acme_ca https://acme-v02.api.letsencrypt.org/directory

    # Rate limiting global
    rate_limit {
        zone global {
            match path *
            events 40
            period 1m
        }
    }

    # Headers de sécurité globaux
    header {
        Strict-Transport-Security "max-age=31536000; includeSubDomains; preload"
        X-Content-Type-Options "nosniff"
        X-Frame-Options "DENY"
        Referrer-Policy "strict-origin-when-cross-origin"
    }
}

# ========================
# Services du stack
# ========================

idm.tundomaine.fr {
    reverse_proxy kanidm:8443 {
        header_up X-Forwarded-For {remote_host}
        header_up X-Forwarded-Proto {scheme}
        header_up Host {http.request.host}
    }
}

cloud.tundomaine.fr {
    reverse_proxy nextcloud:8080
}

mail.tundomaine.fr {
    reverse_proxy stalwart:8080
}

chat.tundomaine.fr {
    reverse_proxy matrix:8008
}

pw.tundomaine.fr {
    reverse_proxy vaultwarden:8080
}

meet.tundomaine.fr {
    reverse_proxy jitsi:8080
}

# Redirection HTTP → HTTPS
:80 {
    redir https://{host}{uri} permanent
}

4. Commandes utiles (Docker)

# Recharger la configuration sans downtime
docker exec caddy caddy reload

# Valider le Caddyfile
docker exec caddy caddy validate

# Voir les logs en temps réel
docker logs -f caddy

# Lister les certificats actifs
docker exec caddy caddy list-certificates

# Tester une URL spécifique
curl -I https://idm.tundomaine.fr
Conseils de sécurité :
  • Change régulièrement l’email ACME
  • Utilise le staging ACME pendant les tests
  • Active le rate limiting adapté à ton usage
  • Surveille les logs Caddy avec Grafana

5. Dépannage courant

  • Erreur 502 Bad Gateway : Vérifier que le service cible tourne et que le port est correct
  • Kanidm ne reconnaît pas les headers : Vérifier les header_up dans le bloc idm.tundomaine.fr
  • Certificat non renouvelé : Vérifier les logs et l’email ACME
  • Rate limit trop strict : Augmenter le nombre d’événements dans le bloc global
