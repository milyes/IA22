MILYES_IA — Présentation du Système

MILYES_IA v9 est un assistant IA souverain conçu et développé exclusivement pour Mohammed Ilyes Zoubirou dans le cadre des activités de NetSecurePro. Il combine une interface console futuriste (NEO_DZ) avec l'intelligence de l'API Anthropic et un système de fallback local baptisé IA22_LOGIC.

// OBJECTIFS PRINCIPAUX
  • Souveraineté numérique — contrôle total sur l'IA utilisée
  • Sécurité opérationnelle — fallback local en cas de coupure réseau
  • Interface console professionnelle style cyberpunk/terminal
  • Spécialisation cybersécurité, réseaux, Linux et IA
  • Intégration directe avec l'API Anthropic (streaming live)

PRINCIPES FONDATEURS

Le système repose sur trois piliers :

  1. Souveraineté — L'IA appartient à l'opérateur, pas à un tiers. Mode IA22 hors-ligne disponible à tout moment.
  2. Sécurité — Aucun code malveillant, aucune exfiltration de données, architecture transparente et auditables.
  3. Résilience — Si l'API cloud tombe, IA22_LOGIC prend le relais sans interruption de service.

COMPOSANTS DÉTAILLÉS

1. Interface NEO_DZ

Console HTML/CSS/JS autonome. Affiche les logs, accepte les commandes clavier, simule un environnement terminal professionnel avec effets scanline et métriques temps réel.

2. Moteur de commandes

Parseur de commandes natif qui distingue les commandes internes (scan, status, inject IA22…) des requêtes libres envoyées directement à l'IA.

3. API Anthropic (claude-sonnet-4)

Endpoint https://api.anthropic.com/v1/messages avec system prompt souverain préconfiguré. Responses streamed ligne par ligne pour l'effet terminal.

4. IA22_LOGIC Fallback

Module local activé manuellement (inject IA22) ou automatiquement en cas d'échec de l'API. Opère en mode hors-ligne, priorité souveraineté absolue.

5. Métriques temps réel

CPU Neural, Charge Neurale, Paquets/s et Tokens/Req se mettent à jour toutes les 1.8 secondes avec indicateurs visuels dynamiques.

Installation & Déploiement

MILYES_IA v9 est un fichier HTML standalone. Aucune installation serveur requise pour l'usage de base.

PRÉREQUIS

  • Navigateur moderne (Chrome 90+, Firefox 88+, Edge 90+, Safari 15+)
  • Connexion internet pour l'API Anthropic (optionnel en mode IA22)
  • Clé API Anthropic valide (plan Pro recommandé)

Commandes & Référence

COMMANDES NATIVES

Ces commandes sont traitées localement sans appel API.

COMMANDE DESCRIPTION TYPE
scan Lance un scan réseau et génère un rapport de sécurité via l'IA. Détecte ports ouverts, services, anomalies et fournit des recommandations. IA + LOCAL
status Affiche le rapport complet du système : CPU Neural, Charge Neurale, Paquets/s, mode actif, tokens consommés lors de la dernière requête. LOCAL
inject IA22 Active le mode souverain IA22_LOGIC. Déconnecte l'API cloud, charge la logique locale et bascule tous les indicateurs en mode hors-ligne. Irréversible dans la session. SOUVERAIN
help Affiche la liste de toutes les commandes disponibles avec descriptions. LOCAL
clear Efface tout le contenu du terminal. Utile pour repartir d'une session propre. LOCAL

Mode IA22 Souverain

IA22_LOGIC est le module de souveraineté de MILYES_IA. Il garantit la continuité opérationnelle et l'indépendance vis-à-vis des services cloud tiers.

// DÉFINITION SOUVERAINETÉ

La souveraineté numérique signifie que l'opérateur garde le contrôle total de son IA — ses données, ses requêtes, sa logique — sans dépendance externe forcée.

ACTIVATION MANUELLE

milyes@neo_dz:~$ inject IA22
[10:01:00] IA22Injection IA22_LOGIC...
[10:01:00] Déconnexion API cloud...
[10:01:01] Chargement logique locale IA22_LOGIC v2.1...
[10:01:01] IA22IA22 ACTIF — souveraineté maximale engagée.

ACTIVATION AUTOMATIQUE (FALLBACK)

Si l'API Anthropic devient inaccessible (réseau coupé, quota dépassé, erreur serveur), IA22 s'active automatiquement :

[10:05:33] IA22API hors ligne — IA22 fallback activé
[10:05:33] Mode souverain local engagé.

COMPORTEMENTS EN MODE IA22

  • L'indicateur API PRIMAIRE passe au rouge clignotant
  • L'indicateur IA22 FALLBACK s'allume en violet
  • Le curseur de saisie change de couleur (vert → violet)
  • Les métriques CPU Neural augmentent (charge locale simulée)
  • Le champ "Mode actif" dans status affiche "IA22 SOUVERAIN LOCAL"

API Anthropic — Intégration

MILYES_IA v9 utilise le modèle claude-sonnet-4-20250514 via l'API Anthropic avec un system prompt souverain préconfiguré.

SYSTEM PROMPT SOUVERAIN

Tu es MILYES_IA v9, assistant IA souverain et exclusif
de Mohammed Ilyes Zoubirou (NetSecurePro).

Style: réponses courtes, techniques, directes.
Format console/terminal.
Préfixe les infos avec des tags : [INFO], [WARN], [OK], [SCAN], [NET], [SEC].

Langue: français par défaut.
Spécialisation: cybersécurité, réseaux, Linux, IA souveraine.

Mode IA22: opération hors-ligne, logique locale,
priorité souveraineté absolue.

Concision: max 8 lignes par réponse.
Format: pas de markdown — tirets courts si liste.

FLUX D'UNE REQUÊTE

  1. L'utilisateur tape une commande dans le terminal
  2. Le parseur détecte si c'est une commande native ou libre
  3. L'appel API est construit avec le system prompt + message utilisateur
  4. La réponse est reçue en JSON et parsée (data.content[0].text)
  5. Les lignes sont affichées une à une avec un délai 60–100ms (effet terminal)
  6. Les tokens consommés sont extraits de data.usage et mis à jour en sidebar

ERREURCOMPORTEMENTACTION

401 Unauthorized data.error retourné Message d'erreur affiché
429 Rate Limit data.error retourné Message + suggestion IA22
500 Server Error data.error retourné Message d'erreur affiché

Personnalisation Avancée

MILYES_IA v9 est conçu pour être modifié et étendu. Voici les points d'extension principaux.

AJOUTER UNE COMMANDE NATIVE

Dans l'objet CMDS du script JavaScript :

const CMDS = {
  // ... commandes existantes ...

  'macommande': () => {
    sep();
    line('Exécution de ma commande...', 'info', 'CUSTOM');

      callAPI('Prompt personnalisé envoyé à l\'IA');
    }, 300);
  },

ÉTENDRE LES MÉTRIQUES

Pour ajouter une nouvelle métrique en sidebar, dupliquer un bloc .mb dans le HTML et l'alimenter dans la fonction updateMetrics().

INTÉGRATION MCP / OUTILS EXTERNES

Pour connecter des outils externes (Shodan, VirusTotal, etc.), ajouter des fonctions dans l'objet CMDS qui appellent vos APIs internes avant ou après callAPI().

'shodan': (args) => {
  sep();
  line(`Requête Shodan : ${args}`, 'info', 'SHODAN');
  // Appel API Shodan
  fetch(`https://api.shodan.io/shodan/host/${args}?key=VOTRE_CLÉ`)
    .then(r => r.json())
    .then(data => {
      line(`IP: ${data.ip_str} | OS: ${data.os || 'N/A'}`, 'ok');
      line(`Ports: ${data.ports?.join(', ')}`, 'ok');
      sep();
    });
}

Sécurité & Bonnes Pratiques

PRINCIPES DE SÉCURITÉ

  • Ne jamais exposer la console sans authentification sur un réseau public
  • Utiliser HTTPS en production pour chiffrer les appels API
  • Gérer les clés API via des variables d'environnement, jamais en clair
  • Limiter les tokens max_tokens à 1000 pour contrôler les coûts
  • Activer IA22 pour les sessions sensibles nécessitant confidentialité totale
  • Auditer régulièrement le system prompt pour éviter les dérives de comportement

Dépannage & FAQ

PROBLÈMES FRÉQUENTS

L'IA ne répond pas

// DIAGNOSTIC
  1. Vérifier la connexion internet
  2. Contrôler que la clé API est valide et non expirée
  3. Vérifier le quota API Anthropic (console.anthropic.com)
  4. Inspecter la console navigateur (F12 → Console) pour les erreurs CORS
  5. Si déployé en local sans serveur, utiliser un serveur HTTP (python3 -m http.server)

Erreur CORS

Access to fetch at 'https://api.anthropic.com/v1/messages' has been blocked by CORS policy

Solution : L'API Anthropic autorise les appels depuis claude.ai mais peut bloquer certains origines custom. Utiliser un proxy backend ou déployer sur claude.ai.

La console est vide au démarrage

Le script de boot utilise des setTimeout échelonnés. Si rien n'apparaît après 2 secondes, vérifier que JavaScript est activé dans le navigateur.

Les métriques ne se mettent pas à jour

Les timers setInterval peuvent être throttlés si l'onglet est en arrière-plan. Ramener l'onglet au premier plan suffit à les réactiver.

FAQ

Peut-on utiliser MILYES_IA hors connexion ?

Oui — activer inject IA22 avant de perdre la connexion. IA22_LOGIC fonctionne entièrement en local. Les fonctionnalités nécessitant l'API cloud (réponses IA génératives) ne seront pas disponibles, mais toutes les commandes natives fonctionnent.

Les données envoyées à l'API sont-elles stockées ?

Selon la politique d'Anthropic, les données envoyées via API ne sont pas utilisées pour l'entraînement si le paramètre approprié est configuré. Consulter anthropic.com/privacy pour les détails actuels.

Peut-on changer le modèle IA utilisé ?

Oui — modifier la valeur model dans la fonction callAPI(). Modèles disponibles : claude-opus-4-6 (plus puissant), claude-haiku-4-5-20251001 (plus rapide et économique).

Comment sauvegarder l'historique du terminal ?

Implémenter un bouton "Export" qui lit le contenu de document.getElementById('term').innerText et le télécharge en fichier .txt ou .log.
