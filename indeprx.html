Langages, Protocoles et Méthodes de Requête de Communication

1. Protocoles de Communication Internet

  • HTTP / HTTPS — Protocole Hypertexte pour la communication web sécurisée.
  • WebSocket — Protocole de communication full-duplex, temps réel.
  • MQTT — Protocole léger pour IoT, messagerie publish/subscribe.
  • TCP / UDP — Protocoles de transport de données, connexion et sans connexion.
  • CoAP — Protocole pour objets connectés (Constrained Application Protocol).
  • gRPC — Framework RPC haute performance basé sur HTTP/2.
  • SOAP — Protocole basé sur XML pour échanges web services.

2. Méthodes HTTP (Verbes de Requête)

  • GET — Récupérer une ressource.
  • POST — Envoyer des données pour créer une ressource.
  • PUT — Remplacer une ressource.
  • PATCH — Modifier partiellement une ressource.
  • DELETE — Supprimer une ressource.
  • HEAD — Récupérer les headers d'une ressource.
  • OPTIONS — Obtenir les options et capacités du serveur.
  • TRACE — Diagnostic de requête.
  • CONNECT — Établir un tunnel vers le serveur (souvent pour HTTPS).

3. Formats de Données de Requête / Réponse

  • JSON (JavaScript Object Notation) — Format léger, lisible et structuré.
  • XML — Format extensible basé sur balises, souvent utilisé avec SOAP.
  • Form Data (application/x-www-form-urlencoded) — Données de formulaire HTML classiques.
  • Multipart/Form-Data — Données multipart pour fichiers et formulaires.
  • Plain Text — Texte brut simple.
  • Protocol Buffers — Format binaire compact Google.

4. Langages et Interfaces de Communication

  • REST API — Architecture basée sur HTTP et méthodes RESTful.
  • GraphQL — Langage de requête pour APIs, flexible et performant.
  • RPC (Remote Procedure Call) — Appels de procédures distantes.
  • gRPC — Implémentation moderne de RPC avec HTTP/2 et Protobuf.
  • Webhooks — Notifications HTTP poussées par un serveur vers un client.

5. Protocoles de Sécurité

  • SSL / TLS — Chiffrement des communications HTTP (HTTPS).
  • OAuth 2.0 — Protocole d’autorisation sécurisé.
  • JWT (JSON Web Token) — Jeton d’authentification sécurisé.
  • API Keys — Clés d’accès pour identification des clients.
